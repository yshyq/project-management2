# 可调主题与项目部署闭环设计

## 目标

1. 前端提供“预设主题 + 自定义主色”的颜色设置。
2. 项目部署按真实人员完成闭环：王二提交，张三接收，张三自行部署或分配李四，实际负责人提交环境信息并完成部署。
3. 自动化测试使用三个真实测试账号验证两条部署路径。

## 主题颜色

- 顶部工具区增加调色板按钮，打开主题设置面板。
- 提供科技蓝、运维绿、清晰红、琥珀金四套预设。
- 提供颜色选择器和 HEX 输入，自定义品牌主色。
- 由主色计算悬停、焦点、选中、按钮阴影与浅色强调背景，保持文字对比度。
- 用户选择保存到 `localStorage`，刷新和重新登录后继续生效。
- 支持恢复默认主题。

## 部署状态机

### 主流程

1. 王二创建项目部署申请，状态为 `待运维接收`，申请人记录为王二。
2. 张三点击接收，状态为 `待安排部署`，记录接收人和接收时间。
3. 张三选择以下任一路径：
   - 点击“我来部署”：当前负责人设为张三，状态为 `部署中`。
   - 选择李四并点击分配：当前负责人设为李四，状态为 `部署中`。
4. 当前负责人填写部署后环境信息并点击“确认部署完成”。
5. 校验通过后状态变为 `已部署`，记录实际部署人与完成时间，同时创建服务器资产记录。

### 权限

- 交付人员可以创建部署申请，不能接收、分配或完成部署。
- 运维 Leader 可以接收待接收申请、自领、分配和在完成前重新分配。
- 运维工程师只能完成当前分配给自己的部署任务。
- 只有当前负责人可以提交部署结果。
- 不允许越级、乱序、重复执行状态动作。
- 所有权限由后端校验，前端按钮显隐仅用于改善体验。

## 部署后环境信息

必填：

- 环境类型
- 内网 IP
- 主机名
- 操作系统
- 用途
- 部署版本

选填：

- 外网 IP
- 备注

部署完成时为每条环境信息创建 `server_assets` 记录，并关联客户、项目名称、部署工单、产品和实际部署人。为了兼容现有项目台账，资产表新增部署工单关联与部署结果字段，不强制先创建传统 `projects` 记录。

## 界面

- 工单详情展示申请人、接收人、当前负责人、实际部署人和状态时间。
- `待运维接收` 时，张三看到“接收”。
- `待安排部署` 时，张三看到“我来部署”和“分配给运维”。
- `部署中` 时，当前负责人看到“提交部署结果”；张三同时可重新分配。
- 部署完成弹窗展示环境信息表单，必填校验通过后才可提交。
- `已部署` 时，详情只读展示部署人和环境信息。

## API

- `POST /api/support-tickets/{id}/receive`
- `POST /api/support-tickets/{id}/assign`，请求 `{ "handlerId": number }`
- `POST /api/support-tickets/{id}/self-assign`
- `POST /api/support-tickets/{id}/complete-deployment`
- `GET /api/users?dept=运维部` 用于分配选择

完成部署请求：

```json
{
  "environment": "生产",
  "innerIp": "10.20.30.40",
  "outerIp": "",
  "hostname": "prod-app-01",
  "os": "Rocky Linux 9",
  "purpose": "应用服务",
  "deploymentVersion": "v2.8.1",
  "remark": "双产品部署完成"
}
```

## 自动化验收

主路径：

1. 登录王二，提交多产品部署申请。
2. 登录张三，接收申请并分配给李四。
3. 登录李四，验证缺少环境必填项不能完成，再填写完整信息完成部署。
4. 验证状态、负责人、实际部署人和服务器资产记录。

分支路径：

1. 登录王二，再提交一张部署申请。
2. 登录张三，接收并点击“我来部署”。
3. 张三填写环境信息并完成。
4. 验证实际部署人为张三。

主题验收：

- 切换预设主题后 CSS 变量变化。
- 自定义 HEX 主色后按钮、焦点和选中态变化。
- 刷新后主题保持。
- 恢复默认主题成功。

## 错误处理

- 状态不匹配返回 `409`，消息说明当前状态不允许该操作。
- 权限或负责人不匹配返回 `403`。
- 环境信息缺失或格式无效返回 `422`。
- 分配目标不是启用中的运维人员返回 `400`。
- 资产写入和工单完成使用同一数据库事务，任一失败全部回滚。
