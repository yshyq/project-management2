# 运维视角项目管理系统开发文档

## 1. 项目概述

本系统以运维管理为核心，服务于项目上线后和交付过程中的支持协作。系统重点不是传统研发项目管理，而是把“项目信息、上线信息、服务器资产、远程方式、部署方式、访问方式、账号凭据、交付支持、研发支持、运维处理、统计分析”统一管理起来。

系统要解决的问题：

- 运维接手项目时信息不完整，服务器、部署方式、访问入口、账号密码散落在聊天记录或个人文档中。
- 交付提出部署、运维支持、Bug、新需求后，责任人和处理进度不透明。
- 支持流程不固定，有时是“交付 -> 运维”，有时是“交付 -> 研发 -> 运维”，还可能按项目配置其他流程。
- 管理层无法快速统计每个项目的支持情况、上线情况、未处理问题和运维工作量。

系统核心目标：

- 建立运维项目台账：统一记录项目经理、客户、远程方式、服务器、部署方式、访问方式、上线状态等。
- 建立支持闭环：交付发起支持，运维和研发按流程处理，最终验证关闭。
- 建立可配置流程：支持不同项目、不同支持类型配置不同处理链路。
- 建立安全凭据管理：各种登录方式和密码必须加密、授权、审计，不能明文暴露。
- 建立用户权限管理：交付和研发默认不能直接查看客户凭证，需要按客户、项目或支持单申请。
- 建立统计看板：按项目、类型、人员、状态、上线情况统计支持工作。

## 2. 用户角色

| 角色 | 主要职责 |
| --- | --- |
| 系统管理员 | 用户、角色、权限、字典、流程模板、系统参数维护 |
| 运维管理员 | 运维项目台账维护、服务器资产维护、上线信息维护、支持单分派 |
| 运维人员 | 处理部署、环境、访问、服务器、故障、巡检等运维支持 |
| 交付人员 | 提交部署申请、运维支持、Bug、新需求、上线协助、客户现场问题 |
| 研发负责人 | 接收需要研发判断的问题，分派研发人员处理 Bug 或新需求 |
| 研发人员 | 处理 Bug、新需求评估、代码问题、版本包问题、技术方案支持 |
| 项目经理 | 查看项目支持情况、上线情况、项目联系人、关键风险和统计数据 |
| 部门负责人 | 查看团队支持量、项目上线情况、未闭环问题和人员工作量 |

## 2.1 登录入口与菜单范围

系统分为两个登录入口：

| 登录入口 | 默认菜单 | 说明 |
| --- | --- | --- |
| 管理员后台管理页面 | 后台首页、项目台账、项目支持、流程配置、产品名称配置、使用类型配置、用户管理、客户凭证管理、运维资产 | 用于查看全部项目支持状态，并配置系统基础数据、流程和权限 |
| 普通用户登录页面 | 工作台、项目台账、项目支持、客户凭证管理 | 用于交付、研发、运维等普通协作人员处理日常支持和申请查看凭证 |

管理员可以在用户管理中配置用户是否可见各菜单。菜单可见性至少包含：工作台、项目台账、项目支持、运维资产、流程配置、产品名称配置、使用类型配置、客户凭证管理、用户管理。

项目台账对所有普通用户可见。涉及客户凭证、账号、密码、密钥等敏感信息时，仅管理员或已授权用户可查看；普通用户可在凭证管理中提交查看申请，默认流转到运维管理员审批。

项目台账中的上线状态、项目经理、交付负责人、运维负责人、研发负责人等关键项目信息，只允许运维管理员或项目创建者更新；其他普通用户仅查看。

## 3. 功能模块

### 3.1 运维项目管理

核心能力：

- 项目台账：维护项目基础信息、项目经理、客户、交付负责人、运维负责人、研发负责人。
- 上线管理：记录上线状态、版本号、部署环境、上线记录、回滚方案，不在工作台强展示上线时间和成功率。
- 项目状态：未上线、上线准备中、已上线、运维中、暂停维护、已下线。
- 项目标签：重点项目、外网项目、内网项目、私有化、SaaS、政企项目等。
- 项目统计：支持单总数、处理中数量、超时数量、Bug 数、新需求数、部署支持数。

建议字段：

| 字段 | 说明 |
| --- | --- |
| 项目编号 | 系统生成，唯一标识 |
| 产品名称 | 产品或项目服务展示名称 |
| 客户名称 | 所属客户 |
| 项目经理 | 项目业务负责人 |
| 交付负责人 | 交付侧联系人 |
| 运维负责人 | 运维侧第一责任人 |
| 研发负责人 | 研发侧支持负责人 |
| 项目状态 | 未上线、上线准备中、已上线、运维中、暂停维护、已下线 |
| 上线状态 | 未申请、待部署、部署中、试运行、上线成功、上线失败、已回滚 |

新建项目台账不录入远程方式、部署方式、访问方式，这些内容归入运维资产和客户凭证管理维护。

### 3.2 项目信息与运维资产

核心能力：

- 客户与联系人：客户名称、联系人、电话、微信/邮箱、客户技术联系人。
- 服务器管理：服务器 IP、内外网地址、系统版本、CPU、内存、磁盘、用途、所属环境。
- 环境管理：开发、测试、预发、生产、灾备等环境。
- 服务组件：数据库、中间件、应用服务、定时任务、第三方依赖。
- 部署信息：部署目录、启动命令、停止命令、日志路径、配置文件路径、备份路径。
- 访问入口：系统访问地址、管理后台、API 地址、监控地址、文档地址。
- 登录方式：SSH、RDP、数据库、后台账号、堡垒机账号、VPN 账号等。
- 凭据管理：账号、密码、密钥、Token 等敏感信息加密保存，按权限查看，记录查看日志。

敏感信息要求：

- 密码、密钥、Token 不能明文存储，数据库只保存加密密文。
- 支持“可见/复制/申请查看”权限控制。
- 每次查看或复制凭据必须写入审计日志。
- 支持凭据有效期、责任人、最后更新时间、是否需要轮换。
- 页面默认隐藏敏感内容，只显示脱敏信息。

### 3.3 项目支持

交付侧可提交的支持类型：

- 部署申请：新项目部署、版本更新、补丁发布、回滚。
- 运维支持：服务器、网络、访问、配置、证书、数据库、备份、日志排查。
- Bug：功能异常、接口异常、数据异常、性能问题。
- 新需求：客户新增功能、报表调整、流程调整、集成需求。
- 咨询协助：环境说明、账号申请、项目资料补充、客户现场协助。

支持单核心能力：

- 创建支持单并关联项目、环境、服务器、版本、附件。
- 支持环境固定为生产、验证、开发，便于交付、研发、运维统一口径。
- 客户名称为项目台账、项目支持、客户凭证、上线记录等表单的首个必填字段。
- 产品名称为产品/项目服务选择项，默认包含 edhr、edhr MAX、MedPro5；业务新增表单中只能选择已有产品名称。
- 新增产品名称由管理员在后台“产品名称配置”中维护。
- 新增项目支持时，客户名称只能从已有客户中选择；产品名称根据所选客户自动筛选。如果客户有多个产品或多套项目服务，可以选择其中一个，不能填写不存在的客户、产品或项目服务。
- 项目台账列表需要支持大量客户下的快速检索：快速搜索、客户名称、产品/服务、负责人、上线状态均使用可输入筛选，输入时匹配已存在的项目台账数据并即时刷新列表。
- 普通用户默认只能看到与自己相关的客户、项目和支持单；管理员可以查看所有人的支持单。
- 支持单需要展示当前处理人；只有流转到当前登录人的支持单才显示处理和转办动作，管理员可查看并介入全部支持单。
- 支持单流转到下一位处理人或被转办给具体人员时，系统自动生成站内消息通知被指派人员。
- 自动按支持类型匹配流程模板。
- 运维可直接处理，也可转办给具体人员、转研发、退回交付、升级负责人。
- 研发处理后可回到运维部署或交付验证。
- 支持优先级、期望完成时间、SLA、超时提醒。
- 支持处理记录、附件、解决方案、复盘结论。

支持单状态：

- 草稿、待运维接收、待研发处理、运维处理中、待交付确认、已解决、已关闭、已驳回、已挂起。

### 3.4 可配置流程管理

系统内置流程：

- 交付 -> 运维：适用于部署、环境、访问、服务器等运维类支持。
- 交付 -> 研发 -> 运维：适用于 Bug、新需求、代码或版本包相关问题。

流程配置能力：

- 按使用类型、产品名称和流程部门配置默认流程，新增流程不绑定客户名称。
- 按支持类型配置流程。
- 使用类型由管理员在后台“使用类型配置”中维护；流程配置和项目支持表单只能选择已有使用类型。
- 流程配置可维护适用部门和默认处理人员，例如默认运维处理人、默认研发处理人、默认交付确认人。
- 提交项目支持时，处理人字段不预设默认值，提单人可以在对应团队范围内选择本次支持单的处理人员；不选择时由系统按项目负责人兜底流转。
- 按优先级、项目标签、客户等级配置流程分支。
- 支持节点角色：交付发起人、运维负责人、运维处理人、研发负责人、研发处理人、项目经理、部门负责人。
- 支持节点动作：提交、接收、分派、转派、退回、处理、验证、关闭、挂起、升级。

### 3.5 用户管理与权限管理

核心能力：

- 组织、用户、岗位、角色、账号状态管理。
- 项目成员维护：项目经理、交付负责人、运维负责人、研发负责人、支持成员。
- 数据权限：本人、本人负责项目、本部门、全部。
- 凭证权限：不可见、申请查看、授权查看、可复制、可维护。
- 凭证管理按客户建档：新增客户凭证时先检索或选择已有客户，再一次性录入远程方式、服务器 IP/端口/账号/密码、数据库 IP/端口/账号/密码、平台登录地址/账号/密码、责任人和已授权用户。
- 凭证类型不由用户手选，系统根据所填信息自动识别：服务器登录信息默认识别为 SSH，3389 端口或 RDP 远程方式识别为 RDP，数据库信息识别为数据库，平台登录地址识别为平台登录，VPN/堡垒机按远程方式补充识别。
- 普通用户默认可见客户凭证管理菜单，可按客户和凭证类型筛选，但不可新增凭证；凭证明文必须按单个凭证类型提交申请，申请默认到运维管理员。
- 运维管理员按客户、产品和凭证类型分别审批；同意后，申请人才可在客户凭证管理和项目台账详情中查看自己有权限的该类型凭证信息。
- 管理员可在客户凭证管理中按项目凭证查看已授权用户，也可按用户反查其拥有的客户、产品和凭证类型权限。
- 所有查看、复制、审批行为进入审计日志。

### 3.6 统计分析

项目维度统计：

- 项目支持单总数、已关闭数、处理中数、超时数。
- 按类型统计：部署、运维支持、Bug、新需求、咨询。
- 按处理方统计：运维处理数、研发处理数、交付退回数。
- 项目上线状态、上线记录数、部署记录数、回滚记录数。
- 项目未闭环问题、长期挂起问题、高优先级问题。

人员维度统计：

- 运维人员处理量、平均响应时长、平均解决时长、超时率。
- 研发人员处理量、Bug 处理数、新需求评估数。
- 交付人员发起量、验证通过率、退回率。

## 4. 权限设计

建议采用 RBAC + 项目数据范围 + 凭据细粒度权限：

| 权限点 | 管理员 | 运维管理员 | 运维人员 | 交付人员 | 研发人员 | 项目经理 |
| --- | --- | --- | --- | --- | --- | --- |
| 创建项目台账 | 是 | 是 | 否 | 否 | 否 | 可申请 |
| 编辑项目信息 | 是 | 是 | 仅项目创建者 | 否 | 否 | 仅项目创建者 |
| 创建支持单 | 是 | 是 | 是 | 是 | 是 | 是 |
| 分派支持单 | 是 | 是 | 负责项目 | 否 | 研发范围 | 本项目可协调 |
| 处理支持单 | 是 | 是 | 被分派 | 否 | 被分派 | 否 |
| 查看服务器信息 | 是 | 是 | 授权项目 | 授权项目 | 授权项目 | 本项目 |
| 查看密码/密钥 | 按凭据权限 | 按凭据权限 | 按凭据权限 | 默认不可见 | 默认不可见 | 默认不可见 |
| 查看统计看板 | 是 | 是 | 本人/本项目 | 本人发起 | 本人处理 | 本项目 |

## 7. 非功能要求

- 安全性：所有接口鉴权，凭据加密保存，敏感操作审计。
- 可追溯：支持单、上线记录、资产变更、凭据查看均记录操作日志。
- 可配置：支持类型、流程模板、项目状态、服务器类型、部署方式可配置。
- 可统计：所有支持单、上线记录、处理动作保留时间字段和处理人字段。
- 可检索：支持按项目、客户、服务器 IP、访问地址、支持单标题快速搜索。

## 8. 迭代计划

### V1 运维台账与支持闭环

- 登录、角色、数据权限。
- 项目台账、项目经理、交付/运维/研发负责人。
- 服务器、部署方式、访问方式、远程方式管理。
- 支持单创建、分派、处理、确认、关闭。
- 内置流程：交付 -> 运维、交付 -> 研发 -> 运维。
- 项目支持统计与上线状态统计。

### V2 凭据与流程增强

- 凭据加密保存、查看授权、审计日志。
- 流程模板配置、按类型匹配流程。
- SLA、超时提醒、问题升级。
- 上线记录、回滚记录、版本记录。

### V3 运维运营分析

- 项目运维驾驶舱。
