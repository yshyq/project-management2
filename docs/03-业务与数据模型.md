# 运维业务与数据模型

## 1. 总体业务闭环

```mermaid
flowchart LR
  A[交付提交支持] --> B{支持类型}
  B -->|部署/环境/访问/服务器| C[运维接收]
  B -->|Bug/新需求/版本问题| D[研发接收]
  D --> E[研发处理]
  E --> F{是否需要运维操作}
  F -->|是| C
  F -->|否| G[交付验证]
  C --> H[运维处理]
  H --> G
  G -->|通过| I[关闭归档]
  G -->|不通过| B
  I --> J[项目支持统计]
  I --> K[知识沉淀]
```

## 2. 可配置流程模型

```mermaid
flowchart TB
  T[支持单创建] --> R[匹配流程规则]
  R --> P1[交付 -> 运维]
  R --> P2[交付 -> 研发 -> 运维]
  R --> P3[自定义流程]

  P1 --> O1[运维接收]
  O1 --> O2[运维处理]
  O2 --> C1[交付确认]
  C1 --> X[关闭]

  P2 --> D1[研发负责人接收]
  D1 --> D2[研发处理]
  D2 --> O3[运维部署/配置]
  O3 --> C2[交付确认]
  C2 --> X

  P3 --> N1[自定义节点]
  N1 --> N2[条件分支]
  N2 --> X
```

## 3. 项目上线状态

```mermaid
stateDiagram-v2
  [*] --> 未上线
  未上线 --> 上线准备中: 创建部署申请
  上线准备中 --> 待部署: 信息确认
  待部署 --> 部署中: 运维开始部署
  部署中 --> 试运行: 部署成功
  部署中 --> 上线失败: 部署失败
  上线失败 --> 待部署: 修复后重试
  试运行 --> 已上线: 验证通过
  试运行 --> 已回滚: 验证失败回滚
  已上线 --> 运维中
  运维中 --> 已下线: 项目停止维护
  已回滚 --> 待部署
```

## 4. 支持单状态流转

```mermaid
stateDiagram-v2
  [*] --> 草稿
  草稿 --> 待运维接收: 提交运维类支持
  草稿 --> 待研发处理: 提交Bug或新需求
  待运维接收 --> 运维处理中: 运维接收
  待研发处理 --> 研发处理中: 研发接收
  研发处理中 --> 运维处理中: 需要部署或配置
  研发处理中 --> 待交付确认: 研发处理完成
  运维处理中 --> 待交付确认: 运维处理完成
  待交付确认 --> 已解决: 验证通过
  待交付确认 --> 待运维接收: 运维问题验证不通过
  待交付确认 --> 待研发处理: 研发问题验证不通过
  已解决 --> 已关闭: 关闭归档
  运维处理中 --> 已挂起: 等待外部信息
  研发处理中 --> 已挂起: 等待外部信息
  已挂起 --> 待运维接收: 恢复到运维
  已挂起 --> 待研发处理: 恢复到研发
  已关闭 --> [*]
```

## 7. 支持单处理时序

```mermaid
sequenceDiagram
  participant D as 交付人员
  participant SYS as 系统
  participant OPS as 运维人员
  participant DEV as 研发人员
  participant PM as 项目经理

  D->>SYS: 创建支持单并选择类型
  SYS->>SYS: 匹配流程模板
  alt 运维类支持
    SYS->>OPS: 推送运维待办
    OPS->>SYS: 接收并处理
  else Bug或新需求
    SYS->>DEV: 推送研发待办
    DEV->>SYS: 处理并提交结果
    DEV->>OPS: 需要部署或配置时转运维
    OPS->>SYS: 完成部署或配置
  end
  SYS->>D: 通知交付确认
  D->>SYS: 验证通过并关闭
  SYS->>PM: 更新项目支持统计
```
