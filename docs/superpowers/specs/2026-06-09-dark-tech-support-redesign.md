# 深色科技蓝业务与界面优化设计

## 目标

修正技术支持、项目需求、其他支持的新建逻辑为“客户 -> 项目 -> 产品”三级联动；移除正式前端和旧静态入口中的“编辑原型”与顶层搜索；将正式前端整体升级为深色科技蓝企业级界面。

## 业务流程

### 项目部署

1. 选择客户。
2. 手工填写项目名称。
3. 选择一个或多个部署产品。
4. 提交后形成该客户可用于后续支持的项目及产品范围。

### 非部署支持

适用于技术支持、项目需求和其他支持：

1. 先选择客户。
2. 系统只加载该客户的已部署项目。
3. 选择项目后，只显示该项目已部署产品。
4. 选择具体产品后才能提交。
5. 切换客户必须清空项目和产品；切换项目必须清空产品。
6. 后端继续校验客户、项目、产品三者关系，不能只依赖前端。

## 界面清理

- 删除根目录旧静态入口中的顶层全局搜索。
- 删除“编辑原型”按钮和对应编辑模式逻辑。
- 删除正式前端中“当前数据不是静态原型”等开发期文案。
- 保留各业务页面自身必要的列表筛选，不删除项目台账和支持单内部查询。

## 视觉系统

已确认方向：深色科技蓝。

- 页面背景：深海军蓝到蓝黑的低对比层次，不使用大面积炫光。
- 侧栏：深蓝固定导航，电光蓝选中态，青蓝作为少量辅助强调。
- 内容区：蓝黑半透明面板与清晰边框，表格保持高信息密度。
- 主按钮：明亮科技蓝；危险操作保持克制红色。
- 圆角：控件 8-10px，主要面板 12-14px。
- 阴影：轻量蓝黑投影，避免厚重悬浮卡片堆叠。
- 字体：优先 Microsoft YaHei、PingFang SC，强化中文标题和表格可读性。
- 动效：仅用于按钮、导航、面板进入与焦点反馈，并支持 reduced-motion。

## 页面结构

- 顶部只保留角色、页面标题、当前用户和退出按钮，不放全局搜索。
- 支持页保留页面级关键字筛选、新建按钮、支持单表格和详情侧栏。
- 新建支持弹窗按照客户、项目、产品的视觉顺序排列，禁用状态和错误提示清晰。
- 工作台、项目、客户、配置、凭证和用户页面复用统一设计令牌，不改变业务信息架构。

## 响应式

- 桌面端维持侧栏、主内容、详情侧栏结构。
- 中等宽度下详情区移至表格下方。
- 手机端侧栏转为可读的两列导航，表单和统计改为单列。
- 表格允许横向滚动，关键按钮与弹窗不溢出屏幕。

## 测试与验收

- 后端：客户级项目查询、项目产品聚合、非法客户项目组合、非法产品组合。
- 前端：客户选择后加载项目、客户切换清空下游、项目切换清空产品、错误与空状态、提交载荷。
- 清理项：正式前端及旧静态入口均不存在“编辑原型”和顶层全局搜索。
- 视觉：桌面和移动端浏览器截图检查，核心流程无控制台错误、请求失败或布局溢出。
- 完整执行 Pytest、Vitest、类型检查、生产构建和 Playwright。
