# 运维项目管理系统文档目录

本目录用于沉淀以运维视角建设的项目管理系统资料，重点覆盖项目台账、项目信息、项目支持、上线情况、服务器资产、部署访问方式和凭证安全管理。

## 文档清单

- [开发文档](./01-开发文档.md)：运维项目台账、项目支持、流程配置、凭证安全、接口与迭代计划。
- [产品方案](./02-产品方案.md)：产品定位、用户场景、页面结构、统计指标、版本规划。
- [业务与数据模型](./03-业务与数据模型.md)：交付/研发/运维流转、上线状态、领域模型、核心数据表。
- [前后端分离开发文档](./04-前后端分离开发文档.md)：前端路由、后端模块、数据模型、接口、权限、安全和部署方案。

## 当前版本范围

第一版建议聚焦运维管理最需要的主流程：

1. 项目信息统一建档，记录项目经理、交付负责人、研发负责人、运维负责人。
2. 服务器、远程方式、部署方式、访问方式统一管理。
3. 交付提出部署、运维支持、Bug、新需求，研发和运维按流程处理。
4. 支持“交付 -> 运维”“交付 -> 研发 -> 运维”和其他可配置流程。
5. 统计每个项目的支持情况、上线情况和未闭环问题。
6. 各种登录方式和密码加密保存、授权查看、审计追踪。
